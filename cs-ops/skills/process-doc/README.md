# cs-ops.process-doc

Creates and updates CS Operations SOPs, governance records, and handoff guides. Produces structured process documentation for CSM handoffs, playbook governance, data quality processes, escalation workflows, segment change procedures, and general SOPs.

## Use it for

- CSM account handoff document
- Playbook governance record
- Data quality SOP
- Escalation workflow documentation
- Segment change process guide
- General CS Ops SOP

## Don't use it for

- Customer-facing documentation
- Onboarding handoff docs (use onboarding.handoff-doc)

## How to trigger it

Say something like:

- "process doc"
- "SOP"
- "handoff guide"
- "document this process"
- "playbook governance"

## What you get

- Structured SOP or governance document (markdown)

## Prerequisites

- cs-ops CLAUDE.md
- Process details from user
