# rev-ops.change-communication-packaging

Packages GTM change decisions (territory, quota, comp, process) into structured stakeholder communications. Produces audience-specific messaging (field, management, finance) with rationale, timeline, and FAQ. All output is [DRAFT] until RevOps lead approves; skill never distributes directly.

## Use it for

- Draft stakeholder communications for territory or quota changes
- Package comp plan updates for field and management audiences
- Produce FAQ and objection-handling supplements for GTM changes
- Format change rationale for board or exec consumption

## Don't use it for

- Sending or distributing communications (output only)
- Making the underlying change decisions
- CRM or system updates

## How to trigger it

Say something like:

- "communicate the change"
- "package the territory change"
- "draft the quota announcement"
- "write the comp rollout comms"
- "change communication"

## What you get

- Audience-specific communication drafts (Markdown, [DRAFT] status)
- FAQ document supplement

## Prerequisites

- Change decision content and rationale
- Target audience list (field / management / finance / exec)
- Timeline for rollout

## Governance

**Approval required** — output must be reviewed before distribution.
- {'G1': 'all ARR/revenue figures flagged [review — not yet a revenue commitment]'}
- All output is [DRAFT] until RevOps lead explicitly approves
- Skill never distributes or posts communications autonomously
