# csm.expansion-onboarding

Builds an expansion onboarding plan for won CSQLs — bridges CSQL close to CSM adoption execution. Produces a structured plan covering new user onboarding scope, milestone calendar, and stakeholder alignment for the expansion cohort.

## Use it for

- Expansion onboarding plan for a won CSQL
- New user onboarding scope for expanded seats or products
- Milestone calendar for expansion cohort

## Don't use it for

- CSQL tracking before close (use rev-ops.csql-tracking)
- Initial onboarding for new accounts (use onboarding domain skills)

## How to trigger it

Say something like:

- "expansion onboarding"
- "onboard expansion"
- "CSQL won"
- "new users onboarding"
- "expansion cohort plan"

## What you get

- Expansion onboarding plan (structured markdown)

## Prerequisites

- csm CLAUDE.md
- Won CSQL details (scope, users, products)

## Governance

- Requires confirmed CSQL close before generating expansion onboarding plan
