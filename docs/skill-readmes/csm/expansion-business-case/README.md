# csm.expansion-business-case

Builds an expansion business case in two modes: csm-led (internal proposal for CSM-driven expansion) or csql (CSQL Qualification Package for Sales handoff). Draws on value delivery history, outcome catalog alignment, and account growth signals.

## Use it for

- CSM-led expansion proposal for internal use
- CSQL Qualification Package for Sales handoff
- Expansion ROI framing from delivered outcomes

## Don't use it for

- CSQL pipeline tracking after handoff (use rev-ops.csql-tracking)
- Expansion onboarding execution (use csm.expansion-onboarding)

## How to trigger it

Say something like:

- "expansion business case"
- "upsell proposal"
- "CSQL"
- "expansion opportunity"
- "build a case for expansion"

## What you get

- Expansion proposal (csm-led mode)
- CSQL Qualification Package (csql mode)

## Prerequisites

- csm CLAUDE.md
- Account value delivery history
- Outcome catalog entries (if configured)

## Governance

- CSQL package requires Sales alignment before formal submission
