# cs-ops.customize

Updates specific sections of the cs-ops company profile without rerunning the full cold-start interview. Displays current values before editing and confirms changes before writing.

## Use it for

- Update a single config section (e.g., health model weights, segment thresholds)
- Reset a section to defaults
- Show current configuration

## Don't use it for

- First-time setup (use cs-ops.cold-start-interview)
- Updates across multiple domains

## How to trigger it

Say something like:

- "update cs-ops config"
- "change health model weights"
- "show current cs-ops settings"

## What you get

- Updated cs-ops/CLAUDE.md

## Prerequisites

- Existing cs-ops/CLAUDE.md

## Governance

- Confirms changes before writing
