# onboarding.onboarding-plan

Generate, update, or summarize the customer-facing onboarding plan document — the primary shared artifact between the CSM and customer team. Defines milestone timeline, ownership per stage, first priorities, communication cadence, and success criteria placeholder. Reads onboarding model, milestone framework, duration targets, and plan format from the onboarding profile.

## Use it for

- Generate a complete plan after kickoff (--draft, default)
- Revise the plan after a milestone completes or scope changes (--update)
- Abbreviated current-status view for async stakeholder updates (--summary)

## Don't use it for

- Sending the plan to the customer
- Modifying PM connector tasks

## How to trigger it

Say something like:

- "onboarding plan"
- "draft plan"
- "update plan"
- "plan document"

## What you get

- Customer-facing onboarding plan document (--draft)
- Revised plan with change log (--update)
- Status summary for async updates (--summary)

## Prerequisites

- onboarding CLAUDE.md (onboarding model, milestone framework, plan format)

## Governance

- Document is a draft — no autonomous delivery to customer
