# onboarding.success-criteria

Define, refine, and track onboarding success criteria with the customer. Produces a structured success criteria document — typically 3–5 measurable outcomes the customer commits to achieving by end of onboarding. Reads criteria format, review cadence, and methodology from the onboarding profile.

## Use it for

- Criteria definition session from scratch (--define, default)
- Adjust existing criteria after scope change (--refine)
- Progress assessment against confirmed criteria (--review)
- Customer-facing summary without internal notes (--export)

## Don't use it for

- Revenue or expansion metrics (use renewals or rev-ops skills)
- Full onboarding plan (use onboarding-plan)

## How to trigger it

Say something like:

- "success criteria"
- "define outcomes"
- "customer outcomes"
- "onboarding goals"

## What you get

- Structured success criteria document (--define)
- Refined criteria with change log (--refine)
- Progress assessment against criteria (--review)
- Customer-facing criteria summary (--export)

## Prerequisites

- onboarding CLAUDE.md (criteria format, review cadence, methodology)

## Governance

- Criteria are 3–5 measurable outcomes; more requires profile override
- Internal review notes excluded from --export output
