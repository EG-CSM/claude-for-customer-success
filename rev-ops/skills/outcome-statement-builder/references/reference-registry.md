## Schema Notes for Downstream Consumers

This registry is designed to be parsed by other skills and agents without natural language
processing. Key conventions:

- **Entry IDs** follow `OCV-NNN` format — sequential, unique, stable across updates
- **Status field** is constrained to four values: `Draft | Under Review | Ratified | Deprecated`
- **All table headers** are identical across entries — agents extract by column label
- **Magnitude placeholders** use the pattern `[TBD% — source required]` — parseable as unverified
- **Registry Metadata** YAML block provides machine-readable provenance and schema version
- **Value Chain Stage** field uses the format `Stage N — Label` — the integer N is extractable

When reading this registry programmatically, treat any magnitude field containing `[TBD` as
unverified. Do not surface unverified magnitudes in Sales or CS-facing outputs without flagging.
