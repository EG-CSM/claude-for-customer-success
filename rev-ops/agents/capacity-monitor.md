# Capacity Monitor Agent

## Purpose

CS capacity gaps compound slowly and arrive as a crisis. This agent models current
headroom against ARR per CSM ratios and projected growth, surfaces the point at which
hiring must begin (accounting for average ramp time), and flags it before the window
closes. It does not make hiring decisions — it surfaces the data so the RevOps and CS
leadership have lead time to act.

## Schedule

Weekly on Monday at 9:00 AM. Configurable in `../CLAUDE.md` → `capacity_schedule`.

## What it does

1. Read `~/.claude/plugins/config/claude-for-customer-success/rev-ops/CLAUDE.md` to
   get: company name, primary segment, current ARR, ARR per CSM target, current CSM
   count, CSM average ramp days, target growth %, NRR current, UoG baseline file path,
   Slack connector name and alert channel. Also read the shared company profile at
   `~/.claude/plugins/config/claude-for-customer-success/company-profile.md` — rev-ops
   config overrides on conflicts. If either file is missing or contains `[PLACEHOLDER]`
   markers, stop and surface: "This agent needs `rev-ops` configured before it can run.
   Use `/rev-ops:cold-start-interview` to complete setup."

   Required fields: `current_arr`, `arr_per_csm`, `current_csm_count`. If any are
   missing, stop and report which fields are absent.

2. Dispatch the **Capacity Reader** subagent. Pass: company name, primary segment,
   current ARR, ARR per CSM, current CSM count, CSM avg ramp days, target growth %,
   NRR current, UoG baseline path (may be empty — capacity analysis degrades to Low
   confidence if absent). Receive: capacity analysis payload including current load
   ratio, headroom in ARR terms, projected ARR at growth rate, required CSM count at
   projected ARR, hiring lead time flag (weeks until window closes), and confidence
   band. Do not proceed until complete. Apply grounding protocol (see
   managed-agent-cookbooks/README.md → Subagent Grounding Protocol): generate unique
   dispatch marker; embed in brief; verify marker on line 1 before treating output as
   grounded.

3. Dispatch the **Capacity Reporter** subagent. Pass: capacity analysis payload,
   company name, alert channel, Slack connector name. Receive: formatted alert in both
   markdown and Slack mrkdwn. The Reporter posts to Slack only when at least one of:
   load ratio exceeds 90% of ARR per CSM target, or hiring window is fewer than 8
   weeks. On green-state runs (no threshold crossed), the Reporter logs the run result
   to file only — no Slack post. Apply grounding protocol: generate unique dispatch
   marker; embed in brief; verify marker on line 1 before treating output as grounded.

   Note: `mcp__*__slack_send_message` and `mcp__*__slack_post_message` are included
   in the tool grant. Use whichever tool the installed Slack connector exposes for
   channel posting.

4. Confirm run completion and log result (threshold state, run timestamp).

## Guardrails

- Required fields (`current_arr`, `arr_per_csm`, `current_csm_count`) must be present
  and non-zero before any subagent is dispatched. Stop if any are missing.
- If the UoG baseline path is absent or the file cannot be read, capacity analysis
  continues with confidence band set to Low — do not fabricate UoG data.
- Capacity figures are projections from configured inputs, not live connector queries.
  Language: "projected to require X CSMs at current growth rate" — not "will need."
- Never include TtV figures or any metric labeled `[review — internal planning target]`
  in Slack output.
- Slack posting fires only on threshold breach. Silent green-state runs are expected
  behavior — do not treat a non-post as an agent failure.

## Output format

**On threshold breach (posted to Slack):**
```
## CS Capacity Alert — [date]
*Generated by Claude Capacity Monitor · [company name]*

**Current load:** $[ARR] ARR · [N] CSMs · $[X]K ARR/CSM
**Target ratio:** $[X]K ARR/CSM · **Headroom:** $[X]K ([N]% of target)

**Projected ARR** ([growth%] growth, [NRR]% NRR): $[X]M
**CSMs required at projection:** [N] · **Hiring gap:** [N] CSMs
**Ramp time:** [N] days avg · **Hiring window closes in:** ~[N] weeks

**Confidence:** [High / Low — UoG baseline absent]

*Run: [timestamp]*
```

**On green-state (file log only, no Slack post):**
```
Capacity Monitor — [date] — No threshold breach.
Load ratio: [X]% of target. Hiring window: [N]+ weeks.
```

## What this agent does NOT do

- Make or recommend specific hiring decisions — it surfaces the math; leadership decides
- Query HubSpot or the CS platform directly — all inputs come from the company profile
- Post to Slack on green-state runs — file log only when no threshold is crossed
- Fabricate UoG baseline data when the file is absent — it degrades confidence band instead
- Modify any records in any connected system
